# Phase 9 MCP Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add secure, persistent stdio and Streamable HTTP MCP tool connections to the existing Electron Agent.

**Architecture:** The official stable MCP TypeScript SDK handles protocol and transports. Focused Main-process modules validate and persist server configs, own connection lifecycles, adapt remote tools into stable `ToolRegistry` snapshots, and gate sensitive calls through Renderer approval IPC.

**Tech Stack:** TypeScript, Node.js 22 LTS, Electron, Vitest, Vite, `@modelcontextprotocol/sdk`, Zod, JSON-RPC, stdio, Streamable HTTP.

## Global Constraints

- Work directly on `main`; do not create a feature branch or use subagents.
- Use the stable `@modelcontextprotocol/sdk`, not beta v2 packages.
- Implement Tools only; do not add Resources, Prompts, Sampling, Elicitation, Tasks, OAuth, legacy SSE, a marketplace, or automatic installation.
- Allow remote `https://` and loopback `http://` only.
- Never expose resolved env/header secrets to Renderer, Agent events, logs, or model context.
- Unknown or ambiguous MCP tools default to `sensitive`.
- Use TDD and keep each commit limited to the named task.
