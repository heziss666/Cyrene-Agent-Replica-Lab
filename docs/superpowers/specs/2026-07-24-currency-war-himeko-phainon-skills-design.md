# 货币战争姬子与白厄阵容 Skill 设计

## 1. 目标

将根目录两份攻略整理为可被 SkillRegistry 自动发现、按需读取的正式 Skill：

- 姬子·启行—列车同行阵容。
- 白厄—以牙还牙甲反伤阵容。

同时把项目中“只适用于 4.4”的错误表述修正为：

> 适用于从 4.4 开始、持续至 4.7 的同一货币战争内容周期；当前结构化数据基线来自 4.4。

本次不修改结构化数据目录名称，不实现货币战争 RAG 或专用查询工具。

## 2. 版本语义

### 2.1 内容周期

货币战争每三个后续版本更新一次。当前玩法、阵容和规则适用范围为 4.4–4.7。

System Prompt 和阵容 Skill 应使用“4.4–4.7 内容周期”，不能再说“只处理 4.4 版本”。

### 2.2 数据基线

`data/currency-war/runtime/4.4/` 表示当前结构化资料在 4.4 建立。只要 4.5–4.7 没有该玩法的独立更新，这份数据继续作为整个周期的基线。

本次不将目录重命名为 `4.4-4.7`，避免破坏导入器、加载器、索引身份和已有测试。未来出现实际玩法更新时，再建立新的数据版本。

### 2.3 证据等级

阵容 Skill 统一使用：

- `cycle-confirmed`：4.4 周期开启时确认，且当前没有 4.5–4.7 更新推翻的事实。
- `historical-guide` 或 `detailed-guide`：攻略中的运营经验，不能替代当前数据。
- `lineup-synthesis`：根据周期数据和攻略形成的阵容判断。
- `needs-validation`：仍需 4.4–4.7 最高难度实战验证。

已有卡芙卡—海瑟音 Skill 中的 `4.4-confirmed`、`4.4-synthesis` 同步改为周期语义。

## 3. 姬子·启行 Skill

### 3.1 标识

```yaml
name: Currency War Himeko Departure Train
description: 当用户在《货币战争》4.4–4.7 标准博弈最高难度中使用姬子·启行作为前台主输出，或需要围绕列车同行、领航员、量子同频进行购买、站位、升星、升级、刷新、装备、止损或转型决策时使用。
version: "1.0.0"
defaultEnabled: true
tools: []
```

目录 ID：`currency-war-himeko-departure-train`。

### 3.2 文件结构

```text
resources/skills/currency-war-himeko-departure-train/
├── SKILL.md
└── references/
    ├── lineup-core.md
    ├── operations.md
    ├── equipment.md
    └── evidence.md
```

### 3.3 `SKILL.md`

只保存：

- 4.4–4.7、标准博弈、最高难度范围。
- 姬子·启行、列车同行、领航员相关触发条件。
- 对局输入检查。
- Reference 路由。
- 本轮唯一主任务。
- 最多 5 个操作、停止条件和后续 1 至 3 节点规划。
- 不得盲目追 6 列车、三星姬子或复制错误装备等硬约束。

### 3.4 `lineup-core.md`

保存：

- 姬子·启行的前台赋能和输出逻辑。
- 领航员的目标选择原则。
- 2、4、6 列车同行档位。
- 4 列车作为稳定目标的原因。
- 量子同频和生存模块。
- 姬子一星、二星、三星的收益和追取条件。
- 前期过渡、中期标准、量子同频混合和后期 6 列车形态。
- 继续投入、仅作过渡和完全转型条件。

### 3.5 `operations.md`

保存：

- 第一位面观察列车成员、姬子和装备信号。
- 第二位面升 7、完成二星姬子和 4 列车。
- 7 级刷新的进入条件与停止线。
- 第三位面在追三星姬子和升级人口之间选择。
- 首领前战力转换。
- 投资环境和策略按经济、刷新、升级、装备、列车成员与即时战力分类。

### 3.6 `equipment.md`

保存：

- 姬子自身装备与领航员复制装备的双层结构。
- 第一件装备、羁绊档位、技能和光轨变化对领航员选择的影响。
- 战场进化手册、步步生花等装备的条件式价值。
- 复制装备不能只看面板强度，必须看复制收益和前后台适配。

### 3.7 `evidence.md`

保存周期内已确认机制、上线后攻略经验、阵容推导和待验证事项。删除作者叙述、网址列表、发布日期和转载说明。

## 4. 白厄反甲 Skill

### 4.1 标识

```yaml
name: Currency War Phainon Counter Armor
description: 当用户在《货币战争》4.4–4.7 标准博弈最高难度中使用白厄作为前台主输出，尤其考虑以牙还牙甲反伤体系，并需要判断能否成型、前排、装备、升星、升级、刷新、止损或转型时使用。
version: "1.0.0"
defaultEnabled: true
tools: []
```

目录 ID：`currency-war-phainon-counter-armor`。

### 4.2 文件结构

```text
resources/skills/currency-war-phainon-counter-armor/
├── SKILL.md
└── references/
    ├── lineup-core.md
    ├── operations.md
    ├── equipment.md
    └── evidence.md
```

### 4.3 `SKILL.md`

只保存：

- 4.4–4.7、标准博弈、最高难度范围。
- 白厄直伤和反甲路线触发条件。
- 对局输入检查。
- Reference 路由。
- 本轮唯一主任务。
- 最多 5 个操作、停止条件和后续规划。
- 没有反甲组件时不能强玩反甲等硬约束。

### 4.4 `lineup-core.md`

保存：

- 白厄“我独自战斗”和“背负希望”机制。
- 直伤白厄与反甲白厄的区别。
- 反甲体系上限高、下限低的原因。
- 护盾核心、高星前台和收尾模块。
- 一件反甲过渡、两件稳定、三件高上限。
- 3 巡海游侠加 2 列车同行等可用骨架。
- 白厄星级和其他成员升星优先级。

### 4.5 `operations.md`

保存：

- 开局先判断装备，不先锁反甲。
- 能玩反甲的基础信号、强烈信号和否决条件。
- 第二位面升 7、刷新白厄和提高前台质量。
- 第三位面在三星白厄、升级人口和第三件反甲之间选择。
- 从直伤转反甲、从反甲退回直伤的条件。
- 敌人和词缀适配。
- 投资策略分类和刷新停止线。

### 4.6 `equipment.md`

保存：

- 以牙还牙甲的核心机制。
- 一、二、三件反甲对应的阵容状态。
- 白厄直伤、启动、防御和护盾装备。
- 反甲装备者选择。
- 不能为了第三件反甲牺牲已成型阵容或最终首领即时战力。

### 4.7 `evidence.md`

保存当前数据、详细攻略、周期验证、阵容推导和待验证事项。删除作者叙述和冗长来源信息。

## 6. 内容整理规则

### 保留

- 阵容机制、角色定位和可替换模块。
- 阵容形态与成型条件。
- 分阶段运营、升级、刷新和升星。
- 装备分配与使用条件。
- 继续、过渡和转型判断。
- 敌人适配、停止线和不确定项。

### 改写

- “作者理解”改为体系原理。
- 固定名单改为核心模块和功能位。
- 固定装备组合改为功能需求与条件式例子。
- 攻略结论改为可执行的条件、优先级和停止线。
- 4.4 单版本表述改为 4.4–4.7 内容周期。

### 删除

- Skill 编写说明。
- 作者视角和创作过程。
- 长篇网址、发布日期、转载说明。
- 重复的范围、模板和禁止事项。
- 与实际决策无关的背景叙述。

## 8. 加载流程

```text
用户提出具体阵容问题
        ↓
模型根据 Skill 名称和描述选择正确体系
        ↓
加载对应 SKILL.md
        ↓
按问题读取 1 至 2 个 Reference
        ↓
结合用户对局总结选出本轮唯一主任务
        ↓
输出最多 5 个操作、停止线和后续节点规划
```

三个阵容 Skill 必须可以同时启用，并依靠明确名称和 description 区分，不使用泛化的 `Currency War Lineup` 或 `DoT Lineup` 名称。

## 9. 测试与验收

- SkillRegistry 能区分并加载三个阵容 Skill。
- 新 Skill 默认启用、可用、`tools: []`。
- 每个新 Skill 恰好有四份 Reference。
- 主文件和 Reference 都不包含原始网址列表。
- 姬子 Skill 保留领航员、2/4/6 列车、7 级刷新和复制装备。
- 白厄 Skill 保留直伤/反甲转换、1/2/3 反甲、护盾前排、敌人适配和 7 级刷新。
- System Prompt 和三个阵容 Skill 使用 4.4–4.7 内容周期。
- 两份根目录原稿不存在。
- Skill 测试、类型检查、全量测试和 Electron 构建通过。

## 10. 非目标

- 不重命名 `data/currency-war/runtime/4.4/`。
- 不新增货币战争 RAG。
- 不新增专用结构化查询工具。
- 不修改 GameState。
