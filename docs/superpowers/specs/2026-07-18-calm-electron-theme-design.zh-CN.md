# Cyrene Agent Calm Workspace 界面改版设计

## 1. 目标

在不改变 Agent、RAG、记忆、Skills、MCP、Scheduler、会话和运行诊断等业务逻辑的前提下，重新设计 Electron renderer 的信息结构和视觉样式。

本次改版需要解决：

- 页面排版不统一，部分内容在常见窗口尺寸下显示不全。
- 顶部工具栏和导航缺少稳定位置，长内容滚动时不方便操作。
- 字体、按钮、输入框、间距和状态样式不统一。
- Chat 与数据管理页面使用相近的布局密度，无法同时兼顾阅读和管理效率。
- Agent Events 长期占据右侧空间，干扰主要任务。

## 2. 设计方向

采用已确认的 `Calm Workspace` 浅色方案：

- 以白色和中性浅灰为主背景，避免大面积单一彩色。
- 使用低饱和绿色表示选中、成功和主要操作。
- 使用细边框和轻微层次区分区域，不依赖大量阴影或装饰卡片。
- 内容圆角主要为 6 至 8 像素；聊天输入框可以使用稍大的圆角。
- 字体使用系统无衬线字体栈，统一字号、行高和字重。
- 图标优先使用 Lucide，陌生图标提供 `title` 或 tooltip。

## 3. 应用框架

主窗口分为三个稳定区域：

1. 左侧导航栏：显示品牌、主要模块、连接状态和设置入口。
2. 顶部页面栏：显示当前页面标题、简短状态和页面级操作。
3. 主内容区：根据页面类型显示聊天内容或数据管理界面。

左侧导航替代现有顶部模块标签，包含：

- Chat
- Memory
- Skills
- MCP
- Tasks
- Runs

当前模块使用浅绿色背景和更明显的文字权重。导航栏在普通桌面窗口固定显示；窗口较窄时折叠为图标栏，避免挤压主内容。

顶部页面栏在主内容滚动时保持可见。它只承载标题、当前会话或页面状态，以及新增会话、Activity 等少量高频操作。

## 4. Chat 页面

Chat 页面以阅读和输入为核心：

- 消息内容限制最大宽度并在主区域居中，宽窗口下不会形成过长文本行。
- 用户消息和 Agent 消息通过背景色、对齐方式和标签区分。
- 工具调用采用紧凑的过程行展示，成功、失败和运行中状态清晰可辨。
- 输入区固定在聊天内容底部，不随历史消息一起消失。
- 输入区保留模型或模式状态、发送按钮和现有必要操作。
- 发送中、取消、错误、重试和空会话状态需要有明确反馈。

输入区不会覆盖最后一条消息，消息列表底部需要预留相应空间。

## 5. 数据管理页面

Memory、Skills、MCP、Tasks 和 Runs 共用一致的数据页面骨架：

- 顶部是搜索、筛选和主要操作。
- 必要时显示少量统计摘要。
- 主体采用紧凑列表或表格，支持快速扫描。
- 详情显示在独立区域或抽屉中，不把完整详情塞入列表项。

各页面保留自己的功能差异：

- Memory：记忆层级、来源、状态、搜索和维护操作。
- Skills：Skill 状态、来源、启用状态和详情。
- MCP：服务器连接状态、工具数量、连接和断开操作。
- Tasks：调度规则、时区、下次运行、立即运行和历史记录。
- Runs：运行状态、耗时、工具调用、Token 信息和事件 Trace。

页面不使用卡片套卡片。表格、工具栏和详情区域通过边框、分隔线和背景层级组织。

## 6. Activity 事件面板

Agent Events 改为默认收起的右侧 Activity 抽屉：

- 顶部页面栏显示 Activity 图标。
- 正常状态下不占据主内容宽度。
- 点击图标后从右侧展开，显示当前运行事件和历史过程。
- 执行失败、等待用户批准或需要处理时，图标显示状态点或数量。
- 抽屉可以关闭，关闭后不影响 Agent 继续运行。
- 窄窗口下以覆盖式抽屉打开，避免同时压缩导航栏和内容区。

## 7. 状态与交互

所有功能页面统一处理以下状态：

- 加载中：稳定占位，不引起布局跳动。
- 空数据：说明当前没有数据，并提供适用的主要操作。
- 错误：显示简短原因和可执行的重试操作。
- 禁用：降低对比度，但仍保持文字可读。
- 成功、警告、失败：分别使用绿、琥珀和红色状态，不只依赖颜色，还显示图标或文字。
- 长文本：允许换行或省略，并可在详情中查看完整内容。

按钮尺寸、输入框高度、焦点样式和键盘操作在所有页面保持一致。

## 8. 响应式规则

主要支持 Electron 桌面窗口，不为手机浏览器重新设计，但需要适应窗口缩放：

- 宽窗口：完整左侧导航、主内容、按需展开的 Activity 抽屉。
- 中等窗口：缩短辅助文字，数据表隐藏低优先级列。
- 窄窗口：左侧导航折叠为图标栏，Activity 使用覆盖式抽屉。
- 所有尺寸下，顶部栏和聊天输入区保持可用。
- 标题、按钮、状态和表格内容不能互相覆盖或超出容器。

## 9. 技术边界

- 本次主要修改 renderer HTML、TypeScript 和 CSS。
- 不改变 main、preload 和 IPC 的业务协议，除非现有界面状态无法通过当前 API 获得。
- 尽量复用现有 DOM 和事件处理逻辑，先调整结构，再统一样式。
- 图标使用 Lucide；若当前项目未安装对应依赖，在实施阶段加入最小依赖。
- 不引入大型 UI 框架，不进行与主题无关的业务重构。

## 10. 验收标准

- Chat、Memory、Skills、MCP、Tasks、Runs 均可从左侧导航访问。
- 顶部页面栏和聊天输入区在滚动时保持稳定。
- Activity 默认收起，可正常展开、关闭并显示异常提醒。
- 原有按钮、表单、IPC 调用和 Agent 交互功能保持正常。
- 常见桌面窗口尺寸下没有内容重叠、控件裁切或不可读文字。
- 页面字号、间距、颜色、边框、按钮和输入控件风格统一。
- 构建、自动化测试和 Electron smoke test 通过。
- 使用桌面与较窄窗口截图进行视觉检查，验证主要页面和状态。
