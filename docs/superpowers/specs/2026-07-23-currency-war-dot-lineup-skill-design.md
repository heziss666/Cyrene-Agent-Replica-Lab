# 货币战争持续伤害阵容 Skill 设计

## 1. 目标

将根目录的 `CURRENCY_WAR_4_4_DOT_LINEUP_SKILL.md` 整理为能够被项目 SkillRegistry 自动发现、按需加载、直接指导 Agent 进行对局决策的正式 Skill。

该 Skill 专门解决《崩坏：星穹铁道》货币战争 4.4 版本、标准博弈、最高难度中的持续伤害（DoT）阵容决策，包括：

- 是否应该选择或继续持续伤害路线。
- 当前节点购买、上阵、站位、升级、刷新和升星的优先级。
- 卡芙卡、海瑟音、黑天鹅等核心角色的培养顺序。
- 2、4、6 持续伤害阵容的过渡与取舍。
- 装备分配、经济止损和转型条件。
- 接下来 1 至 3 个节点的规划。

本次只整理和接入 Skill，不实现货币战争 RAG、专用数据查询工具或自动修改 GameState。

## 2. 文件结构

```text
resources/skills/currency-war-dot-lineup/
├── SKILL.md
└── references/
    ├── lineup-core.md
    ├── operations.md
    ├── equipment.md
    └── evidence.md
```

### 2.1 `SKILL.md`

只保存高频且每次触发都需要的内容：

- Skill 的触发范围。
- 输入信息检查。
- 按局面选择本轮唯一主任务的决策流程。
- 按需读取 Reference 的规则。
- 回答结构、操作数量和停止条件。
- 最关键的禁止事项。

目标是让 Agent 读完 `SKILL.md` 后知道“应该怎么分析”，而不是一次性背下全部攻略。

### 2.2 `references/lineup-core.md`

保存阵容知识：

- 持续伤害体系的输出循环。
- 卡芙卡、海瑟音、黑天鹅、椒丘、艾丝妲、桑博、千冶·刃的定位。
- 前期 2 持续伤害、中期 4 持续伤害、后期功能模块和条件式 6 持续伤害。
- 核心角色升星顺序。
- 继续路线、浅度过渡和完全转型的判断条件。

### 2.3 `references/operations.md`

保存运营知识：

- 第一、第二、第三位面的阶段目标。
- 什么时候停留在 6 级刷新。
- 什么时候升级到 8、9 级。
- 什么时候补即时战力、保经济或完成关键升星。
- 刷新停止线、生命值止损和首领前最低成型标准。
- 投资环境和投资策略只按效果分类，不把旧攻略名称写成固定答案。

### 2.4 `references/equipment.md`

保存装备知识：

- 全队装备分配总原则。
- 卡芙卡至少需要有效装备的硬约束。
- 海瑟音、黑天鹅、椒丘、艾丝妲和千冶·刃的装备方向。
- 装备组合使用“功能需求”表达，不把单一三件套写成唯一答案。
- 区分多波次、小怪和首领环境中的装备价值。

### 2.5 `references/evidence.md`

保存可信度边界：

- `4.4-confirmed`：当前版本已确认事实。
- `historical-guide`：只能借鉴运营骨架的旧攻略。
- `4.4-synthesis`：基于当前资料形成的策略推断。
- `needs-validation`：仍需最高难度实战验证的结论。

该文件保留必要的依据摘要和不确定项，不保留长篇网址清单、发布日期叙述和编写过程说明。

## 3. Skill 元数据

正式 Skill 使用项目现有 frontmatter：

```yaml
---
name: Currency War DoT Lineup
description: 当用户在《货币战争》4.4 标准博弈最高难度中考虑、运营或调整持续伤害阵容，并需要购买、站位、升星、升级、刷新、装备、止损或转型建议时使用。
version: "1.0.0"
defaultEnabled: true
tools: []
---
```

`defaultEnabled: true` 使该 Skill 默认进入可用目录；具体是否加载仍由模型根据名称和描述判断。

本次使用 `tools: []`。当前 `search_knowledge` 只装载旧 Cyrene 知识，不能可靠提供货币战争实体数据，因此 Skill 不应要求调用无关知识库。未来接入货币战争 RAG 或专用查询工具后，再单独更新工具声明和事实查询流程。

## 4. Agent 决策流程

### 4.1 判断是否适用

以下情况触发：

- 用户明确选择或询问持续伤害、DoT、卡芙卡、海瑟音、黑天鹅体系。
- 用户当前阵容已经有持续伤害核心，需要判断是否继续投入。
- 用户需要在持续伤害路线和其他阵容之间转型。

如果用户讨论其他阵容且没有 DoT 取舍，本 Skill 不应主导回答。

### 4.2 检查输入

优先读取用户提供的对局总结，检查会改变决策的关键信息：

- 当前节点和下一战斗节点。
- 生命值、等级、经验和货币。
- 上阵、后台、备战席和角色星级。
- 当前商店。
- 装备库存与分配。
- 羁绊、投资环境、投资策略和已解锁顾问。

信息不完整时只追问关键缺口，并给出条件化建议，不能因为缺少次要字段而拒绝分析。

### 4.3 选择本轮主任务

每次回答只能确定一个主要任务：

- 保经济。
- 补即时战力。
- 完成关键升星。
- 升级人口。
- 寻找海瑟音。
- 寻找黑天鹅。
- 完成 4 持续伤害。
- 条件允许时尝试 6 持续伤害。
- 止损或转型。

其他建议必须服务于该主任务，不能同时给出相互冲突的“继续存钱”和“花光刷新”等指令。

### 4.4 输出执行顺序

最多给出 5 个按实际执行顺序排列的操作。涉及刷新时必须说明：

- 目标角色或目标提升。
- 最多投入多少资源。
- 达成什么条件立即停止。
- 什么情况下提前止损。

回答最后规划接下来 1 至 3 个固定节点，并明确可能改变计划的条件。

## 5. 内容删改规则

### 5.1 保留

- 4.4 版本变化及其对阵容的实际影响。
- 角色定位、阵容形态、运营节奏、升星和装备原则。
- 有明确条件和停止线的决策规则。
- 历史攻略中仍可复用的运营骨架。
- 当前仍不确定、必须降低结论强度的事项。

### 5.2 改写

- “编写者理解”改为不带作者视角的体系原理。
- 长篇解释压缩成决策条件、优先级或反例。
- 固定角色名单改为核心模块与可替换功能位。
- 固定装备套装改为装备功能需求，并保留有条件的高优先级装备。
- 投资策略名称改为按经济、升级、刷新、即时战力等效果分类。
- 来源等级改为 Agent 可以直接用于表达置信度的规则。

### 5.3 删除

- 面向人类读者的 Skill 编写说明。
- 重复出现的版本范围和禁止事项。
- 长篇资料来源网址、发布日期和转载说明。
- 与对局决策无关的创作过程、免责声明和来源故事。
- 已由系统 Prompt 或代码固定、且不需要在 DoT 决策中重复的通用玩法介绍。

## 6. 原始文件处理

正式 Skill 验证通过后删除：

- `CURRENCY_WAR_4_4_DOT_LINEUP_SKILL.md`
- `CURRENCY_WAR_GAMEPLAY_RULES_FOR_AGENT.md`

第二份文件不被运行时代码读取。删除前检查其独有且仍有效的标准博弈规则是否已经存在于以下位置之一：

- 货币战争 System Prompt。
- `src/main/currency-war/rules/`。
- `data/currency-war/runtime/4.4/`。
- 新的 DoT Skill。

没有进入运行时且仍对 DoT 决策必要的内容，应先迁移到对应 Reference。

## 7. 加载流程

```text
用户提出持续伤害阵容问题
        ↓
SkillRegistry 向模型提供 Skill 名称和描述
        ↓
模型调用 Skill 加载能力读取 SKILL.md
        ↓
SKILL.md 根据问题要求读取 1 至 2 个相关 Reference
        ↓
Agent 结合用户对局总结选择本轮主任务
        ↓
输出不超过 5 个操作、停止线和后续节点规划
```

详细资料不应全部同时加载。例如：

- 问“是否升 8”时读取 `operations.md`，必要时再读 `lineup-core.md`。
- 问“装备怎么分”时读取 `equipment.md`，必要时再读对应角色定位。
- 问“这个结论可靠吗”时读取 `evidence.md`。

## 8. 错误与冲突处理

- 用户输入与 Skill 资料冲突时，优先指出版本或状态冲突，不擅自覆盖用户提供的对局事实。
- Skill 与当前结构化数据冲突时，以当前 4.4 结构化数据为事实依据，Skill 中的策略结论降级为参考。
- 对局信息不足时使用条件式建议。
- 不确定结论必须标为推断或待验证。
- 不得把桑博当作普通商店默认角色进行规划，除非用户状态明确显示已解锁。
- 不得为了凑 6 持续伤害牺牲关键生存位或已成型主核心。

## 9. 测试与验收

### 9.1 结构测试

- SkillRegistry 能扫描到 `Currency War DoT Lineup`。
- frontmatter 可解析，`defaultEnabled` 为 `true`。
- `tools` 为空，不错误声明 `search_knowledge`。
- 四个 Reference 文件均存在且非空。
- `SKILL.md` 对四个 Reference 的引用路径正确。

### 9.2 内容测试

- Skill 明确限制为 4.4、标准博弈、最高难度。
- 包含输入检查、本轮唯一主任务、最多 5 个操作和刷新停止线。
- 保留卡芙卡、海瑟音、黑天鹅、椒丘及 2/4/6 持续伤害的核心知识。
- 保留继续投入、浅度过渡和完全转型条件。
- 保留证据等级和待验证事项。
- 不含冗长 URL 列表或“编写者”叙述。

### 9.3 删除测试

- 两份根目录原始文档均不存在。
- 全量 Skill、TypeScript 和 Electron 构建测试不回归。
