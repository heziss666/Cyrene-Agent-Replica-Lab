# 货币战争建议的证据约束设计

## 目标

货币战争 Agent 在回答角色、羁绊、装备、投资环境、投资策略和阵容运营问题时，只能使用项目本地资料中能够找到的事实，或基于这些事实作出的明确推导。资料没有记录的内容必须标记为未知，不能使用模型预训练记忆补全。

## 当前问题

当前 `search_knowledge` 只装载 Cyrene 世界观资料。货币战争结构化数据虽然已经加载为 `CurrencyWarCatalog`，但没有注册为 Agent 工具。阵容 Skill 也只依赖模型主动调用，因此模型可以跳过查询并直接生成答案。

## 证据分层

1. **结构化事实**：角色、羁绊、装备、投资环境和投资策略。唯一事实来源是 `data/currency-war/runtime/4.4`。
2. **攻略经验**：阵容成型条件、运营节奏、装备选择和止损条件。来源是匹配的货币战争 Skill 及其 Reference。
3. **策略推导**：结合当前对局状态、结构化事实和攻略经验产生的本轮建议。推导不得伪装成固定规则。

## 数据流

收到用户消息后，系统先构建本轮证据包：

1. 在用户文本中识别本地目录存在的角色、羁绊、装备、投资环境和投资策略名称。
2. 根据“白厄/反甲”“姬子·启行”“卡芙卡/海瑟音”等路线词自动选择对应 Skill。
3. 读取所选 Skill 的主体和与当前问题相关的 Reference。
4. 再扫描 Skill 内容中出现的结构化实体，补充其精确基础数据。
5. 将基础事实、攻略内容和来源标签组成只读证据包，加入本轮 System Message。
6. 模型只能根据证据包回答；需要查询证据包外实体时调用结构化查询工具。

## 结构化查询工具

新增 `lookup_currency_war_data`：

- 输入：一个或多个名称，可选是否包含关联实体。
- 输出：紧凑的结构化记录和数据版本。
- 精确名称优先，别名和部分名称只用于候选查找。
- 未找到、字段为空或资料未记录时明确返回未知，不做推测。
- 角色只返回费用、站位、职责、羁绊和已记录强化信息。
- 装备返回类型、属性、效果和已记录配方。
- 羁绊、环境和策略返回各自已有字段。

精确游戏事实不使用向量 RAG。向量检索以后只用于攻略文档等非结构化长文本。

## Skill 自动路由

第一版只覆盖项目当前三个阵容 Skill：

- 白厄、反甲、反伤、以牙还牙甲：`currency-war-phainon-counter-armor`
- 姬子·启行、发车、列车体系：`currency-war-himeko-departure-train`
- 卡芙卡、海瑟音、持续伤害、DoT：`currency-war-kafka-hysilens-dot`

一次最多自动加载一个主阵容 Skill。用户明确比较多个体系时可以加载多个，但设置总字符上限，避免上下文失控。

## Prompt 约束

- 涉及游戏实体的事实必须来自本轮证据包或工具结果。
- 不允许凭模型记忆补充羁绊、费用、定位、效果、配方、概率和规则。
- `null`、缺失字段和未命中均视为“本地资料未记录”。
- 建议中的关键动作要简短标注依据类别：基础库、攻略 Skill 或策略推导。
- 证据冲突时以结构化基础库作为实体事实来源，并明确指出攻略可能过期。
- 没有足够证据时给出条件式建议或直接说明无法判断。

## 错误处理

- 证据包构建失败时不阻塞聊天，但 System Message 会明确声明本轮基础数据不可用，禁止输出具体游戏事实。
- Skill 加载失败时仍可根据基础库回答事实，但不能声称使用了对应阵容攻略。
- 查询工具不会返回内部文件绝对路径，只返回稳定来源标签和版本。

## 测试

- 证明当前对局中的阿格莱雅、爻光、乱破和藿藿返回正确羁绊与定位。
- 证明“长线利好”的 `null` 效果不会被补写成经济效果。
- 证明白厄反伤问题会自动加载白厄 Skill。
- 证明证据包同时包含用户输入实体和 Skill 引用实体。
- 证明查询工具的精确命中、候选命中、未知字段和数量上限。
- 证明 System Prompt 明确禁止无来源事实，并要求关键建议标注依据。
- 保持现有 Agent、Skill、RAG、Electron 和完整测试集通过。
