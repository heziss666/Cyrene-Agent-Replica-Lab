# 奖励、补给与遭遇

## 适用前提

- 当前玩法是 4.4 标准博弈。
- 用户正在比较奖励、补给或遭遇候选。
- 候选中的角色、装备、环境与策略效果必须从结构化 JSON 查询。
- 本文件提供评估顺序，不提供固定选项榜单。

## 统一评估顺序

```text
能否解决下一战的明确问题
→ 能否完成当前阵容的确定性提升
→ 是否与已投入路线契合
→ 是否保留后续转型空间
→ 长期上限和稀有度
```

高稀有度、高费用和高理论上限不能自动排在第一位。

## 奖励节点

## 第一步：识别当前缺口

### 生存缺口

- 前台频繁倒下；
- 缺少护盾、治疗或控制；
- 下一战是首领；
- 生命压力高。

### 输出缺口

- 最近战斗接近行动值上限；
- 主输出没有装备；
- 关键羁绊未激活；
- 核心仍为一星；
- 缺少收尾伤害。

### 经济缺口

- 路线已确认但后续需要升级或刷新；
- 当前战力稳定；
- 剩余节点多；
- 经济奖励不会延误下一战。

### 路线缺口

- 尚未出现明确核心；
- 已有两个可能体系；
- 缺少能决定路线的角色、星徽或组件；
- 某个选择能让一条路线立即形成，而另一条仍只是理论。

## 第二步：比较奖励类型

### 可立即升星的角色

若升星后能直接上场、补羁绊或承担核心功能，价值通常很高。

### 关键装备或组件

只有在以下条件下提高优先级：

- 当前有明确穿戴者；
- 可以立即合成；
- 能解决输出或生存；
- 不需要多个未知后续组件；
- 不会让其他核心失去必要装备。

### 阵容关键角色

区分：

```text
拿到即可成型
vs
只是最终阵容的第一张
```

前者通常更有当前价值。

### 通用即时战力

路线未定或生命压力高时，高星过渡、生存和辅助可能优于专属高费核心。

### 经济资源

阵容稳定、下一节点风险低且剩余节点多时，价值上升。

## 奖励节点常见错误

- 只看费用和稀有度；
- 为未来核心放弃立即二星；
- 选择无法使用的装备；
- 忽略备战席和装备栏容量；
- 不看下一节点；
- 把阵容 Skill 的终局名单当成当前选择顺序。

## 补给节点

补给通常同时包含角色与装备，必须评估整组价值。

## 角色部分

检查：

- 是否可立即升星；
- 是否可立即上场；
- 是否补前台、后台、生存或输出；
- 是否达到羁绊关键档；
- 是否与当前等级和后续刷新方向一致；
- 是否只是高费一星挂件；
- 即使转型是否仍有通用价值。

## 装备部分

检查：

- 当前核心能否使用；
- 位置属性是否匹配；
- 是否为关键合成组件；
- 是否需要拆装资源；
- 是否会挤占更重要装备；
- 是即时强度还是长期组件；
- 效果是否依赖当前不存在的羁绊或角色。

## 补给整体比较

可为每组标注：

```text
即时战力：高 / 中 / 低
路线契合：高 / 中 / 低
转型弹性：高 / 中 / 低
装备可用性：高 / 中 / 低
升星价值：有 / 无
下一战价值：高 / 中 / 低
```

优先选择总收益更稳定的整组，而非只看最亮眼的一件。

## 路线未确定时的补给

优先：

- 可立即二星的角色；
- 通用生存；
- 多体系可用辅助；
- 可承接现有装备的输出；
- 通用装备；
- 同时补两条潜在线路的成员。

谨慎：

- 只能服务单一终局且短期无法上场的高费角色；
- 需要多件后续组件才生效的装备；
- 会迫使卖掉当前高星过渡的套餐。

## 遭遇节点

遭遇是生命资源投资，不只是奖励选择。

## 风险信号

提高风险前检查：

- 当前生命值；
- 最近两场战斗结果；
- 是否接近行动值上限；
- 核心是否稳定；
- 前台是否有生存；
- 装备是否已经分配；
- 遭遇后距离首领多远；
- 奖励是否解决明确缺口；
- 敌人是否克制当前体系。

## 更偏向低风险的条件

- 阵容尚未成型；
- 最近普通战斗吃力；
- 生命压力高；
- 下一节点接近首领；
- 高难奖励只是理论上限；
- 缺少敌方或词缀信息；
- 当前路线仍需时间发育；
- 失败会影响评价或直接出局。

## 可以提高风险的条件

- 阵容已经稳定；
- 最近战斗有明显余量；
- 生命健康；
- 奖励能立即完成关键升星、装备或路线；
- 遭遇后仍有补强节点；
- 已知敌人不克制当前阵容；
- 当前必须提高后期上限才能通过最终首领。

## 遭遇难度不是固定答案

早期社区攻略常建议阵容未成型时选择较低难度，可作为保守框架。

但不得写成：

```text
所有遭遇永远选最低难度
```

也不得写成：

```text
奖励越高越应该选最高难度
```

必须结合当前局面。

## 与阵容 Skill 的关系

若阵容 Skill 与本文件产生表面冲突：

- 精确实体事实以 JSON 为准；
- 成熟阵容关键组件优先参考 Skill；
- 本文件用于比较当前节点风险和即时价值；
- 最终建议结合 GameState 推导。

例如：

```text
Skill说某装备是终局核心
≠
当前补给一定选择该装备
```

如果下一战会失败，能立即完成二星和生存的另一组补给可能更优。

## 信息不足时如何追问

### 奖励

- 三个候选分别是什么？
- 购买后能否立即升星或补羁绊？
- 下一节点是什么？

### 补给

- 每组包含哪些角色和装备？
- 当前核心、站位和装备缺口是什么？
- 是否有拆装资源？

### 遭遇

- 当前生命与最近战斗结果？
- 各难度奖励和敌人是什么？
- 遭遇后距离首领还有几步？

## 不可当作硬规则的经验

- 奖励永远优先经济；
- 补给永远优先高费；
- 遭遇永远选低难或高难；
- 专属核心装备出现就无条件选择；
- 第一位面只拿低费、第三位面只拿高费；
- 套餐中有五星角色就一定最好。
