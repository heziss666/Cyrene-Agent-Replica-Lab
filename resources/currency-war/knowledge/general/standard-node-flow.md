# 标准博弈节点流程

## 适用前提

- 当前玩法是 4.4 标准博弈。
- 当前问题涉及节点顺序、当前节点能做什么、下一节点准备或位面节奏。
- 具体奖励、敌人、商店、投资效果与角色效果必须从当前 GameState 和结构化 JSON 查询。
- 本文件只提供固定流程和节点决策框架，不预测随机内容。

## 固定节点顺序

### 第一位面

```text
1-1 奖励
→ 1-2 奖励
→ 选择投资策略
→ 1-3 战斗
→ 1-4 战斗
→ 1-5 补给
→ 1-6 战斗
→ 1-7 遭遇
→ 1-8 奖励
→ 1-9 首领
```

### 第二位面

```text
2-1 战斗
→ 选择投资策略
→ 2-2 战斗
→ 2-3 补给
→ 2-4 战斗
→ 2-5 遭遇
→ 2-6 奖励
→ 2-7 首领
```

### 第三位面

```text
3-1 战斗
→ 选择投资策略
→ 3-2 战斗
→ 3-3 补给
→ 3-4 战斗
→ 3-5 遭遇
→ 3-6 奖励
→ 3-7 首领
```

## 硬规则与局内变量

可以直接依赖：

- 当前位面的固定节点顺序；
- 下一固定节点的类型；
- 每个位面末端为首领；
- 第三位面首领后结束本局。

不能由本文件推断：

- 奖励节点具体候选；
- 补给中的角色与装备；
- 遭遇难度和奖励；
- 当前商店内容；
- 战斗结果；
- 投资环境或策略造成的特殊覆盖。

若当前效果明确写有“替换”“不再”“改为”等语义，应先查询 JSON 并应用覆盖，再解释节点。

## 每个节点需要更新的信息

```text
当前位面
当前节点编号
当前节点类型
下一节点类型
小队生命值
金币、等级、经验
前台、后台与备战席
商店与锁定状态
装备和可拆装资源
当前羁绊与下一档缺口
投资环境与投资策略
最近战斗结果
可见敌人、首领和词缀
```

缺少下一节点或敌方信息时，只能给条件化建议。

## 奖励节点

### 决策目标

先确定当前最缺的资源：

- 可立即完成升星；
- 可立即形成关键羁绊；
- 核心装备组件；
- 当前缺少的前台、后台、生存或输出；
- 经济与后续操作空间；
- 尚未确定路线时的通用过渡能力。

经验排序：

```text
立即生效的确定提升
> 补关键缺口的路线资源
> 与当前装备、站位和费用匹配的未来资源
> 短期无法使用的纯理论上限
```

阵容 Skill 可以覆盖该经验排序。

### 常见错误

- 只因费用或稀有度更高就选择；
- 只看最终阵容，不看下一战；
- 选当前无法穿戴或无法合成的装备；
- 为未来可能出现的体系，放弃当前可立即升星的主力。

## 投资策略节点

必须先从 JSON 查询候选完整效果，再评估：

- 即时收益还是延迟收益；
- 对下一战是否生效；
- 是否要求特定角色、装备、羁绊或消费行为；
- 剩余节点能否回收；
- 是否与当前环境、路线冲突；
- 选择后是否需要重新评估阵容路线。

不能将高稀有度直接等同于更好。

## 普通战斗节点

战前依次检查：

1. 免费站位调整；
2. 免费装备调整；
3. 能立即完成的升星；
4. 能立即达到的羁绊档位；
5. 升级后是否有有效角色填入新位置；
6. 有明确目标与止损线的刷新。

战后必须记录：

- 胜负；
- 生命值变化；
- 是否接近行动值上限；
- 哪个前台过早倒下；
- 主输出是否正常启动；
- 哪些角色或装备没有发挥作用。

## 补给节点

角色与装备作为整组评估：

- 角色能否立即上场；
- 是否完成或接近升星；
- 是否补关键羁绊；
- 站位是否补当前缺口；
- 携带装备是否适合现有核心；
- 整组是否提高下一战胜率；
- 即使后续转型是否仍有通用价值。

不要只因其中有一名高费角色就选择整组。

路线未确定时，优先：

- 高星过渡；
- 通用生存；
- 现有角色能立即使用的装备；
- 同时保留两条可能路线的组合。

## 遭遇节点

同时考虑：

- 当前生命值与评价压力；
- 阵容是否成型；
- 最近战斗是否接近失败；
- 遭遇后距离首领还有多少节点；
- 奖励能否解决明确缺口；
- 高难敌人是否克制当前阵容；
- 失败后是否仍有生命止损。

阵容未成型、最近战斗吃力或生命压力高时，降低遭遇风险通常更合理。

这不是“永远选最低难度”的硬规则。阵容稳定、生命健康且奖励能显著提高上限时，可以提高风险。

## 位面首领

首领前即时战力权重提高。

建议顺序：

1. 修正站位、装备和错误羁绊；
2. 完成确定性升星；
3. 补入可立即发挥作用的角色；
4. 比较升级人口与刷新；
5. 设置刷新目标和停止条件；
6. 再决定保留多少经济。

不得为了长期经济放弃明显影响首领胜率的确定提升，也不得无目标花光资源。

## 各位面目标

### 第一位面

- 用自然商店和奖励建立低成本过渡；
- 保持路线弹性；
- 建立能处理普通敌人的输出与生存；
- 记录核心角色、装备组件和投资信号；
- 不为未确认终局过早投入全部资源。

### 第二位面

- 根据角色进度、装备与羁绊确认主路线；
- 完成关键核心稳定星级；
- 将过渡阵容升级为中后期稳定阵容；
- 在升级、刷新和保经济中选定本节点主任务；
- 首领前不能继续依赖只有理论上限的未成型阵容。

### 第三位面

- 提高阵容上限；
- 完成剩余核心升星、关键人口与装备；
- 根据最终首领加入对策；
- 降低延迟收益优先级；
- 最终首领前把资源转为确定战力。

## 需要查询 JSON 的事实

- 角色费用、站位、羁绊和赋能；
- 当前等级商店概率；
- 装备效果与配方；
- 投资环境和策略；
- 羁绊档位；
- 专家顾问解锁；
- 当前版本调整。

## 不可当作硬规则的经验

- 某固定节点必须花到固定金币；
- 奖励永远选经济；
- 补给永远选高费；
- 遭遇永远选最低或最高难度；
- 首领前必须花光；
- 所有阵容采用同一升级节奏。
