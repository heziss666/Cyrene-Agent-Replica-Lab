## 2. 玩法核心概念

| 术语 | 含义 | Agent 需要关注的状态 |
|---|---|---|
| 对局 | 从开始到生命值归零或最终通关的一次完整运行 | 模式、难度、位面、节点、生命值 |
| 位面 | 一局共三个大阶段，每个位面有多个节点和首领 | `plane = 1/2/3` |
| 节点 | 对局推进的最小阶段，可包含战斗、奖励、遭遇、补给、选择等 | 节点类型、序号、下一节点 |
| 小队生命值 | 对局级生命值，不等同于战斗中单个角色生命值 | 当前值、是否需要止损、结算评价阈值 |
| 行动值/倒计时 | 战斗必须在限制内清除敌人 | 是否能在限时内取胜 |
| 金币 | 招募、买经验、刷新商店的核心经济资源 | 当前金币、战后预计金币、利息档位 |
| 等级/团队规模 | 等级决定可上阵角色数量，并改变商店费用概率 | 当前等级、经验、升级成本、目标等级 |
| 商店 | 招募角色的主要入口，每节点自动刷新一次，可手动刷新或锁定 | 商店候选、是否锁定、刷新成本 |
| 星级 | 同名同星级角色合成后的强度等级 | 各角色持有数量、升星缺口 |
| 站位 | 前台、后台、前后台；正确区域才能激活赋能 | 各角色所在区域、是否放对位置 |
| 羁绊 | 上阵一定数量同羁绊成员后激活分级增益 | 当前人数、下一档阈值、边际收益 |
| 投资环境 | 通常是整局级规则修改器或开局/长期增益 | 当前环境及其所有规则修正 |
| 投资策略 | 在固定节点选择的即时、延迟或持续规则修改器 | 已有策略、触发进度、剩余节点 |
| 专家顾问 | 默认不在普通商店，需通过特定入口解锁的特殊角色 | 是否解锁、解锁来源、是否已进入商店池 |
| 优势布局 | 跨对局解锁的永久/赛季增益 | 账号已解锁布局，不能混入本局临时策略 |

## 3. 一局的完整流程

### 3.1 对局开始前

1. 选择模式：标准博弈或超频博弈。
2. 标准博弈可选择已解锁的职级难度；难度越高，敌人和词缀越强。
3. 加载账号级“优势布局”和当前赛季规则。
4. 进入本局，获得初始小队生命值、初始资源和初始阵容/投资环境。
5. 记录当前版本、模式、职级、优势布局，避免后续将不同模式规则混用。

**确定规则：** 一局包含三个位面，每个位面面对不同敌人及首领。[S1][S2]

**需补证据：** 标准和超频模式各自的精确初始金币、初始等级、初始商店和初始角色数量，公开资料未形成可靠的全版本统一表。

### 3.2 每个节点的通用循环

```text
进入节点
  → 结算“进入节点时”触发的环境/策略/羁绊效果
  → 商店自动刷新（除非锁定或被规则禁止）
  → 玩家进行准备操作
  → 节点专属选择/奖励/战斗
  → 战斗或事件结算
  → 更新金币、连胜、生命值、经验、策略计数器
  → 进入下一个节点
```

准备阶段可能包含：

- 招募或出售角色；
- 购买经验、提升等级和上阵人数；
- 手动刷新或锁定商店；
- 调整前台/后台站位；
- 上阵、下阵或整理备战席；
- 穿戴、合成或拆卸装备；
- 检查羁绊档位、升星缺口和下一节点风险。

### 3.3 位面结束与整局结束

- 每个位面末端需要击败首领后进入下一位面。
- 第三位面最终首领胜利后完成本局。
- 小队生命值降至 0 时立即失败。
- 标准博弈通关评价由剩余小队生命值决定。

## 4. 节点类型与决策意义

公开规则可以确认节点类型和作用，但**没有找到一份可证明适用于 4.4 所有模式的固定完整节点序列表**。投资环境和策略还可能替换节点，因此 Agent 不应把“1-2 必定是什么”写死。

| 节点类型 | 已确认作用 | 玩家可做的主要事情 | Agent 的关键问题 |
|---|---|---|---|
| 普通战斗 | 在有限行动值内击败敌人；失败扣小队生命值 | 战前整队、决定是否花钱提升即时战力 | 当前阵容能否稳胜？是否值得破利息止损？ |
| 奖励节点 | 战斗难度相对较低，提供金币、角色、晶矿或其他奖励 | 以较低风险积累资源 | 是否可以保留经济，不为低风险节点过度刷新？ |
| 遭遇节点 | 存在不同难度和奖励；版本公告明确提到难度 3/4 | 选择风险/收益，完成战斗或事件 | 生命值和战力是否允许选高难？ |
| 补给节点/补给阶段 | 从多组选项中取得角色和装备；优势布局可增加一个选项 | 选择最匹配阵容的组合、合成装备 | 核心输出缺什么装备？是补战力还是补体系？ |
| 投资策略节点 | 从候选策略中选择，部分优势布局允许各刷新一次 | 选择即时或长期策略 | 剩余节点能否回收延迟收益？策略是否改变后续操作？ |
| 投资环境节点 | 选择对整局生效的环境，可能决定开局阵容或规则 | 确定本局主方向 | 环境要求追什么羁绊/角色/经济路线？ |
| 首领节点 | 位面结算战；进入首领时可能触发专属策略 | 最后强化、检查装备和羁绊 | 当前强度是否达到通关线？是否应全部花钱？ |
| 特殊/转换节点 | 环境或策略可把节点替换、增加或改变奖励 | 按修改后的规则行动 | 必须先应用所有规则修改器，再判断节点含义 |

### 4.1 节点不能写死的原因

- 版本会调整遭遇节点和奖励节点。[S11]
- 环境可能把普通战斗替换为遭遇节点。
- 策略可能在若干节点后触发奖励、禁止商店操作或改变刷新内容。[S10]
- 超频博弈每个位面节点更少。[S7]

因此状态中应直接记录当前界面显示的 `node_type` 和 `node_index`，规则引擎只解释该节点，不自行猜测完整路线。

## 5. 小队生命值、战斗胜负与评价

### 5.1 生命值规则

- 正式教程数据表明，初始小队生命值为 **100**。[S5]
- 在战斗、遭遇和首领节点取胜时，小队生命值保持不变。[S5]
- 未在限定行动值内消灭全部敌人即视为未胜利，并扣除一定小队生命值。[S1][S5]
- 小队生命值降至 0 时本局结束。[S1][S5]

### 5.2 标准博弈职级评价

| 通关剩余小队生命值 | 职级提升 |
|---:|---:|
| `>= 70` | `+3` |
| `>= 40 且 < 70` | `+2` |
| `< 40` | `+1` |

挑战失败不会降低当前职级。[S2]

### 5.3 Agent 的生命值决策原则

这些属于运营原则，而不是固定最优解：

1. **生命值是可消耗资源，但也是通关评价资源。** 高血量时可以接受少量风险换长期经济；接近 40/70 阈值时，额外保血会直接影响职级提升。
2. **首领前应提高止损权重。** 剩余投资回收时间短，金币的未来利息价值下降，即时战力价值上升。
3. **不知道扣血公式时不能伪造预测。** 第一版 Agent 应表达为“低/中/高风险”，直到通过实测建立伤害表。

### 5.4 尚未确认的关键数值

- 各节点失败时的精确扣血公式；
- 扣血是否由敌方剩余单位、剩余生命、节点难度或职级共同决定；
- 战斗行动值上限在不同节点/词缀下的计算方式。

这些应通过游戏内连续实测补齐，不应让 LLM 猜测。

## 6. 金币、利息、经验与商店

### 6.1 金币用途

金币可用于：

- 招募 1-5 费角色；
- 购买经验、提升等级和团队规模；
- 手动刷新商店。[S1][S2]

### 6.2 战斗后收入

```text
本节点金币收入 = 基础金币 + 连胜金币 + 利息
```

每持有 **10 金币**，战斗结束时获得 **1 金币利息**，利息最多 **5 金币**。[S4]

因此标准模式下的利息断点为：

| 战斗结算前金币 | 利息 |
|---:|---:|
| 0-9 | 0 |
| 10-19 | 1 |
| 20-29 | 2 |
| 30-39 | 3 |
| 40-49 | 4 |
| >=50 | 5 |

### 6.3 经济决策的直接规则

1. 计算一次购买/刷新后是否会跌破下一个十金币断点。
2. 在强度足以过节点时，保留 50 金币通常具有最高持续利息收益。
3. 当预计失败损失大于未来利息时，应允许“破利息”提升战力。
4. 临近最终首领时，未来节点减少，持币的利息价值下降。
5. 超频模式**不获得利息**，不能沿用标准模式的“存 50”策略。[S7]

### 6.4 等级与上阵人数

- 使用金币购买经验可提升等级。
- 等级对应可上阵角色数量。[S1][S2]
- 教程实际流程显示，升至 5 级后可新增一个上阵位置；但完整 1-10 级经验需求表仍需游戏内采集。[S12]

### 6.5 商店规则

- 每个节点自动刷新一次商店。
- 可花费金币手动刷新。
- 可锁定商店；锁定后下个节点不会自动刷新。[S1][S2]
- 商店等级越高，高费用角色概率越高，低费用角色概率越低。[S2][S9]

### 6.6 商店刷新概率

| 等级 | 1费 | 2费 | 3费 | 4费 | 5费 |
|---:|---:|---:|---:|---:|---:|
| 1 | 100% | 0% | 0% | 0% | 0% |
| 2 | 100% | 0% | 0% | 0% | 0% |
| 3 | 100% | 0% | 0% | 0% | 0% |
| 4 | 65% | 25% | 10% | 0% | 0% |
| 5 | 45% | 33% | 20% | 2% | 0% |
| 6 | 30% | 40% | 25% | 5% | 0% |
| 7 | 19% | 30% | 40% | 10% | 1% |
| 8 | 18% | 25% | 32% | 22% | 3% |
| 9 | 15% | 20% | 25% | 30% | 10% |
| 10 | 5% | 10% | 20% | 40% | 25% |

来源：[S2]

### 6.7 费用对应的刷新等级建议

这是官方教程型运营建议，可用于 Skill，但不应视为绝对规则：

- 2 费主输出阵容，可在 6 级集中刷新；此时 2 费概率为 40%。
- 4 费主输出阵容，应尽快升至 8 级再集中寻找；8 级 4 费概率为 22%。[S9]

对于其他费用，可以依据概率表自动计算“当前费用出现概率最高的等级”，再结合剩余节点和升级成本做选择。

### 6.8 仍需确认

- 每次手动刷新的基础金币成本及所有修正规则；
- 各等级经验需求、一次购买经验获得多少经验；
- 各节点基础金币和连胜金币表；
- 不同星级角色的出售价格与退款公式；
- 商店固定格数及其被策略修改时的规则。

## 7. 角色、站位、赋能、升星与羁绊

### 7.1 角色费用

角色分为 1-5 费。费用越高，基础战斗能力通常越强；同费用角色基础强度近似，但功能和体系价值不同。[S1][S2]

Agent 不能只按费用比较，必须同时考虑：

- 是否是当前阵容核心；
- 是否补到关键羁绊档位；
- 是否能立即升星；
- 是否符合前台/后台位置；
- 是否有合适装备；
- 是否会挤占更关键的上阵位或备战席。

### 7.2 站位和角色赋能

角色站位分为：

- 前台；
- 后台；
- 前后台。

角色放在正确区域时，激活对应角色赋能。赋能可能强化原有技能或增加新技能。[S1][S2]

因此 Agent 在提出阵容时必须输出明确位置，而不能只列角色名称。

### 7.3 升星

- 角色初始为 1 星。
- 招募 3 个相同星级的同名角色后自动升星。
- 3 星通常是常规最强形态。[S1][S2]

为支持购买判断，状态中至少需要能够推导：

```text
当前场上数量 + 备战席数量 + 商店可购买数量
```

例如已有两个 1 星角色，商店出现第三个时，其即时价值远高于普通单张。

### 7.4 羁绊

- 上阵达到指定人数后激活羁绊。
- 人数越多，羁绊档位和效果通常越强。[S1][S2]
- 羁绊效果可能不只是战斗数值，也可能提供金币、装备或战后奖励，因此必须读取你已导入的 `bonds.json`，不能只按“战力加成”处理。

Agent 应计算：

1. 当前已激活羁绊；
2. 当前人数与下一档阈值；
3. 加入/移除某角色后失去或新增的羁绊；
4. 主羁绊、辅助羁绊和独立羁绊是否抢占位置；
5. 羁绊收益是否值得牺牲角色星级或单体强度。

## 8. 装备规则

### 8.1 基础规则

- 装备提供属性和特殊战斗效果。
- 主要由奖励节点、补给节点获得。
- 两件简易装备可合成一件进阶装备。
- 进阶装备继承两件简易装备的数值属性，并获得独特效果。[S1][S2]
- 每名角色最多穿戴三件装备。[S8]
- 已穿戴装备通常需要拆装扳手等机制拆取。[S2]

### 8.2 Agent 的装备分配原则

1. 先确定核心输出/核心机制角色，再分配稀缺进阶装备。
2. 区分“基础属性适配”和“特殊效果适配”。
3. 检查前台强度、后台强度等属性是否与角色实际位置一致。
4. 合成前比较：保留两件简易装备分别穿戴，还是合成为一件进阶装备更有价值。
5. 未拥有拆装能力时，错误穿戴会产生机会成本。

### 8.3 第一版 Agent 可采用的输出格式

```text
装备操作：将【高周波电锯】给【核心前台角色】
原因：该角色位于前台，能利用前台强度，并且是当前主要输出。
优先级：高
替代方案：若尚未合成，则暂时分别保留两件简易装备。
```

## 9. 投资环境、投资策略与规则修改器

### 9.1 投资环境

- 在固定节点选择。
- 对整局提供增益或修改规则。[S1][S2]
- 可能直接提供初始角色/装备、提高特定羁绊刷新概率、增加补给、替换节点、解锁专家顾问等。[S10]

投资环境应在 Agent 中被视为**高优先级全局上下文**。读取状态后，先应用环境，再评价商店和阵容。

### 9.2 投资策略

策略可以分为以下决策类型：

| 类型 | 例子 | Agent 要记录的内容 |
|---|---|---|
| 即时资源 | 立即获得金币、经验、角色、装备 | 选择后立即更新状态 |
| 持续触发 | 每次进入节点、战斗结束或购买若干角色后触发 | 计数器、触发条件 |
| 延迟收益 | 若干节点后获得奖励 | 选择节点、剩余等待节点 |
| 商店修改 | 免费刷新、全特定费用、禁止购买/刷新 | 当前商店规则覆盖 |
| 节点修改 | 转换奖励/遭遇节点，增加未来选择 | 节点地图/下一节点类型 |
| 生命值交易 | 扣血换资源、按失血给予奖励 | 当前血量和评价阈值 |
| 阵容增益 | 指定站位、羁绊或角色获得增益 | 目标角色和激活条件 |
| 装备修改 | 合成触发、装备重随机、获得武装箱 | 装备库存和触发次数 |

投资策略页面明确包含银色、金色、棱彩三个稀有度，并标注可出现位面 1/2/3。[S10]

### 9.3 规则覆盖顺序

建议程序按以下顺序计算：

```text
基础模式规则
→ 当前职级/敌人词缀
→ 优势布局（跨局）
→ 投资环境（整局）
→ 已有投资策略（整局/定时）
→ 羁绊与角色赋能
→ 当前节点临时规则
```

发生冲突时，优先使用游戏内明确的“替换/不再/改为”描述；不要由 LLM 自行决定覆盖关系。

## 10. 专家顾问

专家顾问的确定规则：

1. 初始不能通过普通商店招募。
2. 可通过特定投资环境、投资策略或专家邀请函在本局解锁。
3. 解锁后，本局内像普通角色一样进入商店：可购买、升星、穿装备。
4. 保留角色原有羁绊。[S6]

### Agent 必须追踪的状态

```text
advisor_unlocked: 是否已经解锁专家顾问池
unlocked_advisors: 已解锁的顾问名称
unlock_source: 环境 / 策略 / 邀请函
```

在未解锁时，不应建议“刷新商店寻找某专家顾问”；应先建议满足其解锁条件。

4.4 调整公告显示专家顾问会随版本扩充，因此顾问名单必须来自当前实体数据，而不是写死在规则代码中。[S3]

## 11. 标准博弈与超频博弈差异

| 规则 | 标准博弈 | 超频博弈 |
|---|---|---|
| 节奏 | 完整流程 | 每个位面节点更少，耗时更短 |
| 难度来源 | 自由选择已解锁职级 | 依据标准模式最高职级开放 |
| 经济 | 有基础金币、连胜金币和利息 | 每节点开始自动获得更多经验和金币，但无利息 |
| 运营重点 | 利息、刷新时机、长期收益 | 快速成型、立即收益，不存利息 |
| 结算奖励 | 标准晋升收益 | 同难度下晋升点和等价钻钞略少 |

来源：[S2][S7]

**程序要求：** 所有与利息有关的建议必须先检查 `mode`。超频模式下应直接关闭利息计算。

## 12. 阵容规划的决策框架

当前不应先写死大量“固定阵容答案”，而应建立可复用的阵容规划 Skill。

### 12.1 一套阵容知识卡应包含

| 字段 | 说明 |
|---|---|
| 阵容名称 | 便于 RAG 检索的中文名称 |
| 核心角色 | 必须追星/给装备的主输出或主机制角色 |
| 主羁绊目标 | 早期、中期、最终档位 |
| 辅助羁绊 | 用较少位置补充的第二体系 |
| 前后台布局 | 每个核心角色的固定站位 |
| 过渡角色 | 核心未到前可承接装备和战力的角色 |
| 目标等级 | 在哪个等级开始集中刷新 |
| 升星优先级 | 哪些角色必须 2/3 星，哪些只凑羁绊 |
| 关键装备 | 核心装备、可替代装备、合成优先级 |
| 投资环境适配 | 哪些环境会强制或明显倾向该体系 |
| 策略优先级 | 经济、刷新、羁绊、装备、生命值策略的排序 |
| 转型条件 | 什么时候放弃当前体系、转向其他体系 |

### 12.2 每回合建议的优先顺序

1. 检查输入信息是否完整。
2. 应用模式、环境、策略等规则修改器。
3. 评估下一节点风险和当前小队生命值。
4. 检查是否能立即升星或达到羁绊关键档位。
5. 检查站位和装备是否明显错误。
6. 决定金币用途：购买角色、升级、刷新、锁店或保留利息。
7. 给出本节点操作顺序。
8. 给出后续 1-3 个节点的目标，而不是假装能确定整局随机结果。

### 12.3 建议输出模板

```text
当前判断：阵容强度中等，下一节点为首领，继续存钱风险较高。

本回合操作：
1. 购买【角色A】，完成二星升级。
2. 将【角色B】从后台移到前台，激活其前台赋能。
3. 将【装备X】给【角色A】。
4. 升至8级后再刷新，最多刷新到剩余20金币。

目标：
- 立即达到【主羁绊】4人档；
- 后续寻找【核心4费角色】；
- 保留【过渡角色】直到核心至少二星。

风险与备选：
- 若商店未出现核心角色，停止刷新并锁定当前可升星单位。
```

## 13. 第一版对局状态至少需要哪些信息

这是下一步状态 Schema 的最小信息清单，本文件暂不规定最终 JSON 命名。

### 必填

- 模式：标准/超频；
- 职级难度；
- 当前位面和节点；
- 当前节点类型、下一节点类型（可见时）；
- 小队生命值；
- 金币、等级、经验；
- 场上前台和后台角色及星级；
- 备战席角色及星级；
- 商店角色和是否锁定；
- 已有装备及穿戴情况；
- 投资环境；
- 已有投资策略；
- 专家顾问解锁状态。

### 最好提供

- 当前连胜数；
- 本节点预计利息；
- 已解锁优势布局；
- 当前敌人/首领及词缀；
- 策略延迟触发剩余节点数；
- 可用拆装、刷新、选择次数等特殊资源。

## 14. Agent 的“信息够不够”检查

在给建议前，Agent 应按问题类型检查必需字段。

| 用户问题 | 最少需要的信息 |
|---|---|
| “商店里买谁？” | 金币、等级、阵容、备战席、商店、主目标羁绊、下一节点 |
| “要不要刷新？” | 模式、金币、利息档位、等级、目标费用、生命值、下一节点 |
| “要不要升级？” | 当前等级/经验/成本、上阵位需求、目标费用、剩余节点 |
| “装备给谁？” | 全队位置、角色赋能、装备、当前核心、可否拆卸 |
| “选哪个策略？” | 三个候选、当前环境、阵容、金币、血量、剩余节点 |
| “怎么转型？” | 全部角色、备战席、商店、装备、已有羁绊、核心星级、剩余资源 |

缺少关键字段时，应先追问一到三个最高价值问题，而不是生成看似具体但没有依据的建议。

## 15. 可直接编码的规则清单

以下规则适合进入确定性模块：

1. 三个位面及生命值归零失败。
2. 前台/后台/前后台位置合法性。
3. 三个同名同星级角色自动升星。
4. 羁绊人数档位计算。
5. 每 10 金币产生 1 利息，最多 5；仅标准模式。
6. 商店 1-10 级费用概率表。
7. 每节点商店自动刷新，锁定时不刷新。
8. 每角色最多穿戴三件装备。
9. 两件简易装备合成进阶装备。
10. 标准模式通关生命值对应职级 +3/+2/+1。
11. 专家顾问未解锁时不在普通商店；解锁后按普通角色处理。
12. 超频模式无利息。

以下内容只适合 RAG/Skill：

- 是否应存 50 金币；
- 某节点是否应该破利息；
- 哪个阵容更强；
- 何时转型；
- 装备和升星优先级；
- 版本攻略中的具体运营节奏。

## 16. 当前仍缺失、建议游戏内采集的规则

优先级 P0：

1. 标准/超频各自完整节点序列及每个节点可选项。
2. 每个节点基础金币、连胜奖励表。
3. 不同节点失败扣血公式。
4. 1-10 级经验需求、购买经验成本与单次经验值。
5. 商店手动刷新成本及特殊情况下的变化。
6. 各星级角色出售价格。
7. 备战席基础容量、物品栏容量和满容量处理规则。
8. 投资环境/投资策略每次出现的候选数量及刷新规则。
9. 补给节点候选数量、奖励池和刷新/替换机制。

优先级 P1：

10. 不同职级敌人词缀的精确效果。
11. 战斗行动值与敌人波次规则。
12. 角色池共享数量、卡池抽取是否存在有限池机制。
13. 星徽、特殊道具、投影仪和聘用书的通用操作规则。
14. 环境与策略冲突时的覆盖顺序。

推荐采集方式：每条规则保存一张游戏内截图，并记录版本、模式、职级、位面、节点和界面位置。

## 17. 接入项目的建议

### 17.1 RAG 文档

将本文按章节切分进入 RAG，元数据建议包含：

```text
knowledge_type: gameplay_rule
version: 4.4
confidence: confirmed / secondary / heuristic / unresolved
topic: economy / shop / combat / node / lineup / strategy / advisor
```

### 17.2 Skill

建立 `currency-war-advisor` Skill，职责应是：

1. 读取手动输入的对局状态；
2. 调用已有实体查询工具；
3. 检查信息完整性；
4. 计算合法操作、利息、羁绊和升星机会；
5. 检索阵容/运营知识；
6. 输出当前操作、后续目标、理由和风险。

### 17.3 确定性代码与 LLM 的边界

```text
代码负责：数值计算、引用校验、合法动作、规则覆盖、状态更新。
RAG负责：玩法解释、阵容知识、过渡思路、版本攻略。
LLM负责：根据目标和上下文权衡，并用自然语言解释。
```

不要让 LLM 自行计算商店概率、利息、升星数量或羁绊人数。

## 18. 结论与下一步

当前五份轻量实体数据解决了“游戏里有什么”；本文解决了“游戏如何运行、哪些变量决定下一步”。

下一步最合理的交付是：

1. 把本文放入项目的规则知识目录，供 RAG 和 Skill 读取；
2. 根据第 13 节设计手动输入的 `GameState`；
3. 编写状态完整性检查器；
4. 实现利息、商店概率、升星、羁绊和位置检查等确定性函数；
5. 再建立阵容规划 Skill；
6. 最后连接 Electron 对局录入与建议面板。
